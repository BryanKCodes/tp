SummonersBook User Guide

SummonersBook is a desktop app for esports players (League of Legends), optimized for a fast Command Line Interface (CLI) with clear visual feedback via GUI. If you can type fast, SummonersBook helps you manage players and form balanced teams faster than traditional GUI apps.

Quick start

  1. Ensure you have Java 17 or above installed on your computer.
    Mac users: Use the precise JDK version prescribed here.
  2. Download the latest summonersbook.jar from Releases.
  3. Copy the file to the folder you want to use as the home folder for SummonersBook.
  4. Open a command terminal, cd into the folder that contains the jar, and run:
    java -jar addressbook.jar
    A GUI should appear in a few seconds (with some sample data).
  5. Try some commands:
    • add n/Faker rk/Grandmaster rl/ADC c/Sivir — Add a player
    • find rl/Support — Filter by role Support
    • group — Auto-create balanced 5‑player teams
    • viewteam 1 — Show details for Team 1
    • exit — Exit the app
  6. Refer to the Features section for full command details.

Features

Notes about the command format:
  • Parameters in UPPER_CASE are supplied by the user.
  • Optional items are in square brackets, e.g., find [n/NAME] [rl/ROLE].
  • Items with … after them can be used multiple times or not at all, e.g., [t/TAG]….
  • Parameters can be in any order. e.g., add rk/RANK n/NAME rl/ROLE c/CHAMPION.
  • Indices start from 1, referring to the currently displayed list.
  • Players already in a team cannot be edited.
  • Case-insensitive matching for applicable parameters.

SummonersBook commands

Viewing help : help

A help window will appear, displaying our User Guide page.

Format: help

Adding a player: add

Format: add n/NAME rk/RANK rl/ROLE c/CHAMPION

Notes:

  • All fields are mandatory.
  • Parameters can be provided in any order.

Examples:

  • add n/Faker rk/Grandmaster rl/ADC c/Sivir
  • add n/Imissher rk/Gold rl/Support c/Thresh

Listing all players : list

Shows a list of all players in SummonersBook.

Format: list

Listing all teams : listteam

Shows a list of all teams in SummonersBook.

Format: listteam

Deleting a player: delete

Removes a player permanently.

Format: delete INDEX

  • Deletes the person at the specified INDEX.
  • The index refers to the index number shown in the displayed person list.
  • The index must be a positive integer 1, 2, 3, …

Example: delete 3

Success example: Deleted Player: Bernice Yu; Rank: Gold; Role: Mid; Champion: Ahri; Tags: [colleagues][friends].

Failure output:

Invalid command format!
delete: Deletes the player identified by the index number used in the displayed player list.
Parameters: INDEX (must be a positive integer)
Example: delete 1

Finding players by keyword : find

Searches for players by keyword(s) in their name. Matching is case-insensitive and based on whole words only.

Format: find KEYWORD [MORE_KEYWORDS...]

Examples:

  • find joanne lim — finds Joanne Koh, Joanne Lim, and June Lim, but not John Kim.
  • find john — finds John Doe and John Smith.
  • find jo — finds Jo Lin, but not “John Doe” or “John Smith”.

Filtering players : filter

Narrows the player list using one or more filters. Matching is case-insensitive.

You can filter by:

  • rl/ — role (exact match)
  • rk/ — rank (exact match)

Format (any order, at least one filter): filter [rl/ROLE ...] [rk/RANK ...]

How filtering works:

  • Filters of different types (e.g. rl/ and rk/) are combined with AND.
  • Filters within the same type (e.g. multiple rl/ values) are combined with OR.
  • Matching is case-insensitive and exact.

Examples:

  • filter rl/Mid Jungle — players who are Mid OR Jungle.
  • filter rl/Mid rk/Gold — players who are Mid AND Gold.
  • filter rl/Mid Jungle rk/Gold Silver — players who are (Mid OR Jungle) AND (Gold OR Silver).

Editing a player : edit

Edits an existing player in your roster. Players already in a team cannot be edited.

Format: edit INDEX [n/NAME] [rl/ROLE] [rk/RANK] [c/CHAMPION] [t/TAG]

Notes:

  • INDEX refers to the number shown in the current displayed player list. Must be a positive integer (1, 2, 3…).
  • At least one field to edit must be provided.
  • Existing values are overwritten by the new input.
  • Tags are replaced, not added. To clear all tags, type t/ with no tag values.

Examples:

  • edit 1 n/John Doe rl/Mid rk/Diamond c/Ahri — updates name, role, rank, and champion
  • edit 2 t/ — clears all tags
  • edit 3 rl/Top rk/Gold — updates role and rank

Auto-grouping players into teams : group

Forms as many balanced teams of five as possible from unassigned players.

Format: group

How it works:

  • Only players not already in a team are considered.
  • Each team has no conflicting roles (each role appears at most once).
  • Players are grouped to create teams with similar ranks whenever possible.
  • Teams are automatically created until there are fewer than five unassigned players remaining.

Manually creating a team : makegroup

Creates a team manually by specifying exactly five players by name. Each player must already exist and must not belong to any other team.

Format: makegroup n/PLAYER1 n/PLAYER2 n/PLAYER3 n/PLAYER4 n/PLAYER5

Notes:

  • All five player names are required.
  • Each name must exactly match an existing player's name (case-insensitive).
  • Players already assigned to a team will cause the command to fail.
  • A new team will automatically be named Team X where X is the next index.

Examples:

  • makegroup n/Faker n/Deft n/Oner n/Ber yl n/Keria
  • makegroup n/Joanne Lim n/Aaron Tan n/Ivan Lee n/Sarah Ong n/Kai Chen

Disbanding a team or all teams : ungroup

Disbands one team (returns its players to the unassigned pool) or disbands all teams.

Format: ungroup TEAM_INDEX|all

Examples:

  • ungroup 1 — disbands the 1st team
  • ungroup all — disbands all teams

Clearing all entries : clear

Clears all entries from SummonersBook.

Format: clear

Exiting the program : exit

Closes the application.

Format: exit

Saving the data

Data is saved automatically to disk after any command that changes data. No manual save is required.

Editing the data file

SummonersBook stores data as a JSON file at:

[JAR file location]/data/summonersbook.json

Advanced users can edit this file directly if needed.

Caution:

  • Editing the data file incorrectly can corrupt your data.
  • This may cause SummonersBook to start with an empty file.
  • Always back up the file before making changes.

FAQ

Q: How do I transfer my data to another Computer?
A: Install the app on the other computer and overwrite the empty data file it creates with the file that contains the data of your previous home folder.

Q: Why can’t I delete a player who’s on a team?
A: Team rosters must always have 5 players. Remove the player from their team (via editteam in future versions or ungroup and regroup) before deleting.

Q: How are teams named when using group?
A: Sequentially (Team A, Team B, …), skipping any existing names to avoid conflicts.

Known issues

  1. Multiple screens: If you move the app to a secondary screen and later switch to only the primary screen, the GUI may open off-screen. Delete preferences.json before running again.
  2. Help Window minimized: Running help while the Help Window is minimized won’t open a new one. Manually restore the minimized Help Window.
